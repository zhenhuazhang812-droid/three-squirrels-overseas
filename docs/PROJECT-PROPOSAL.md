## 一、项目背景与目的

### 1.1 企业背景

**三只松鼠股份有限公司**（证券代码：300783）是中国坚果零食行业龙头企业，成立于2012年，总部位于安徽合肥。公司产品涵盖坚果、干果、肉脯、烘焙等十余个品类，连续多年蝉联中国坚果销量冠军。

随着"一带一路"政策深化和跨境电商快速发展，三只松鼠积极布局海外市场，已进入美国、加拿大、澳大利亚、新加坡、马来西亚、泰国、越南等20余个国家和地区的零售渠道。

### 1.2 网站建设目的

| 维度 | 目的 |
|------|------|
| **品牌展示** | 建立三只松鼠海外专业形象，展示企业实力与产品优势 |
| **B2B获客** | 吸引海外经销商、批发商、连锁零售商主动询盘 |
| **渠道赋能** | 为海外销售团队提供标准化物料，提升谈判效率 |
| **内容沉淀** | 积累品牌故事、产品资料、认证资质等数字资产 |
| **竞争壁垒** | 区别于低价竞争对手，以品牌力支撑溢价空间 |

### 1.3 战略意义

```
海外采购商认知路径：
  搜索品牌 → 官网验证 → 社交媒体旁证 → 主动询盘
  
官网是不可替代的"信任锚点"：
  - 官网有 → 证明企业正规、有实力
  - 官网无 → 海外客户信任度立即下降
```

## 二、目标客户与需求分析

### 2.1 核心客户画像

#### 客户类型A：区域分销商
- **特征：** 有当地零售网络，熟悉进口流程
- **诉求：** 价格政策、MOQ门槛、产品清单、物流方案
- **决策周期：** 1-4周

#### 客户类型B：连锁零售商（超市/便利店）
- **特征：** 议价能力强，注重合规资质
- **诉求：** FOB报价、认证文件（FDA/HALAL/KOSHER等）、退货政策
- **决策周期：** 1-3个月

#### 客户类型C：电商平台卖家
- **特征：** 关注利润空间和爆款单品
- **诉求：** 支持一件代发、有竞争力的拿货价、侵权风险低
- **决策周期：** 1-2周

#### 客户类型D：餐饮/食品服务商
- **特征：** 批量采购，注重食品安全
- **诉求：** SGS/CIQ检测报告、工厂认证、产品规格稳定性
- **决策周期：** 2-4周

### 2.2 客户核心需求优先级

```
🥇 第一层（必须满足）
  ├── 清晰的产品目录与规格说明
  ├── 明确的价格政策（MQL/FOB/EXW）
  ├── 真实的认证文件展示
  └── 便捷的询盘渠道

🥈 第二层（增强信任）
  ├── 专业的企业介绍与工厂实拍
  ├── 品牌故事与市场数据
  ├── 合作案例与客户见证
  └── 清晰的联系方式与响应承诺

🥉 第三层（差异化加分）
  ├── 多语言支持
  ├── 交互式产品筛选
  ├── 在线客服支持
  └── 订单追踪系统
```

### 2.3 客户痛点洞察

| 痛点 | 解决方案 |
|------|---------|
| 无法判断供应商真实实力 | 首页核心数据（年销售额/覆盖国家/合作客户） |
| 产品选择困难 | 智能筛选系统（按品类/认证/MOQ/价格） |
| 担心语言沟通障碍 | 5种语言实时切换 |
| 担心货不对板 | 视频中心真实展示生产与产品 |
| 不知道如何开始合作 | 清晰的"如何合作"引导页面 + 表单询盘 |

### 3.1 网站现状

| 加载速度 | ⭐⭐⭐⭐ | 静态页面表现良好 |

### 3.2 关键问题清单

**P0 - 影响询盘转化**
- [ ] Hero区域视频加载慢，部分地区无法播放
- [ ] 联系表单字段过多，海外客户习惯简单表单
- [ ] 没有明确的"最小起订量"和"价格区间"展示

**P1 - 影响品牌形象**
- [ ] 部分产品图片为示意图，非真实产品照片
- [ ] 缺乏工厂/仓库实拍图，影响B端信任
- [ ] About页面内容单薄，无差异化卖点

**P2 - 影响运营效率**
- [ ] 更新产品需要改代码，无法独立运营
- [ ] 无访问数据分析，无法优化
- [ ] 无在线客服，客户流失

## 八、技术架构建议

### 8.1 当前架构（静态网站）

```
现状：
  HTML + CSS + JS → GitHub → Netlify CDN
  
优点：简单、稳定、零成本
缺点：更新需要代码操作，无法独立运营
```

### 8.2 推荐架构（渐进升级）

```
Phase 1（保持现状，补充工具）：
  Notion（内容管理） + GitHub → Netlify
  
Phase 2（引入CMS）：
  Strapi/WordPress（自托管） → GitHub → Netlify
  或：Contentful + Next.js 静态生成
  
Phase 3（完整电商）：
  Shopify/woocommerce + Netlify
  支持：在线支付、订单管理、库存同步
```
